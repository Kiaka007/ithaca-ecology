# sp_academy/library/notes — Notes

## Mode: Notes

**Rule:** Ask questions before answers.

This is a space for thinking and slow sense-making. Understanding emerges here through different formats and registers of thought.

## Structure

Each subdirectory carries a different epistemic posture:

- **field/** — Preserve experience, not interpretation. "What did I actually observe?"
- **methods/** — Practical wisdom. What worked, what failed, how to do things.
- **research/** — Slow synthesis, careful claims. Ideas that have been tested.
- **readings/** — What texts have changed my thinking? Engagement with others' ideas.
- **threads/** — Evolving concepts with no deadline or deliverable. See `threads/CLAUDE.md`.

## Conventions

**Field notes preserve raw experience.** Interpretation happens elsewhere.

**Research notes make careful claims.** If uncertain, it belongs in the journal or `bag/meta/` first.

**Readings engage with source material.** Not summaries — dialogue with texts.

## Shell Commands

| Command | Effect |
|---------|--------|
| `notes` | Go to notes root |
| `field` | Go to field/ |
| `methods` | Go to methods/ |
| `research` | Go to research/ |
| `readings` | Go to readings/ |
| `readx <source-name>` | Create/open a reading note |
| `fieldx <study-name>` | Create a new field study with pre/raw/post phases |
| `methodx <topic>` | Create/open a method note |

## Working Here

- Let ideas stay fuzzy in the journal before migrating to research
- Field notes should be writeable in the moment — brevity is fine
- Methods are for future-you: write what you wish past-you had written
- When a note outgrows notes/, it may become a thread
