# sp_academy/library/notes/threads — Threads

## Mode: Threads

**Prompt:** Where can this grow?

Threads carry thought, not obligation. They are evolving concepts that can be paused or dissolved without failing or being abandoned. Unlike commissions, threads have no deliverable — only development.

## Structure

Each thread is a directory containing:

- **decisions/** — Choices made along the way. Records of forks taken.
- **handoffs/** — Transfer documents when authority moves to another context.
- **feedback/** — Responses received. What came back from sharing.
- **returns/** — What came back from handoffs. Artifacts that return to the thread.

Threads may also contain an `overview.md` or other files as the concept develops.

## Lifecycle

Threads are not tasks. They do not need to be completed.

- A thread can be **active** (being developed)
- A thread can be **paused** (set aside, may return)
- A thread can be **dissolved** (released without judgment)
- A thread can **migrate** to `workshop/commissions/` if it acquires obligation and a deliverable

## Shell Commands

| Command | Effect |
|---------|--------|
| `threads` | Go to threads root |
| `threadx <name>` | Create new thread with subdirs |
| `status [hours]` | See threads touched recently |

## Working Here

- Let threads evolve at their own pace
- Document decisions so future-you understands the path taken
- Use handoffs when work crosses to another context
- Returns capture what comes back — feedback, revised artifacts, lessons
- If a thread starts feeling like obligation, it may want to become a commission
