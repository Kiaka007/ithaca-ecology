# AI Layer Architecture in Ithaca

*Originally written as supporting material for the OpenMined lightning talk, April 2026.*

---

## Overview

Four file types govern how Claude Code operates within Ithaca's information ecology.

```
CLAUDE.md files  ── Orientation & norms       ── designed once by human
Memory files     ── Persistent context         ── written by Claude
Skill files      ── Invocable rituals          ── designed once by human
Agent files      ── Session artifacts          ── auto-generated by Claude Code
```

## Flow: What Happens at Runtime

```
Human opens session in a sphere directory
          │
          ▼
  ┌───────────────────────────────────────────────────┐
  │  1. READ ~/.claude/CLAUDE.md                      │
  │     "You are context-dumb. Read the sphere        │
  │      CLAUDE.md before acting."                    │
  └───────────────────────┬───────────────────────────┘
                          │
                          ▼
  ┌───────────────────────────────────────────────────┐
  │  2. READ sphere/CLAUDE.md                         │
  │     learns: role · NPC · tone · norms             │
  │     learns: which memory files to read            │
  └───────────────────────┬───────────────────────────┘
                          │
                          ▼
  ┌───────────────────────────────────────────────────┐
  │  3. READ sphere/.memory/ files                    │
  │     active threads · patterns · blockers          │
  │     + global ~/.claude/.../memory/ (user profile) │
  └───────────────────────┬───────────────────────────┘
                          │
                          ▼
  ┌───────────────────────────────────────────────────┐
  │  4. WORK with human                               │
  │     research · drafting · synthesis · planning    │
  └──────────┬─────────────────────────┬──────────────┘
             │                         │
             │  human invokes skill    │
             ▼                         │
  ┌──────────────────────┐             │
  │  READ                │             │
  │  <sphere>/.claude/    │             │
  │  commands/<skill>.md │             │
  └──────────────────────┘             │
                                       ▼
  ┌───────────────────────────────────────────────────┐
  │  5. WRITE on departure                            │
  │     sphere/.memory/ ── updated threads, patterns  │
  │     global memory   ── user prefs, project state  │
  │     agents/         ── session logs (auto)        │
  └───────────────────────────────────────────────────┘
```

## Key Design Decisions

**CLAUDE.md files are hierarchical** — Claude arrives knowing nothing. The cascade
`global → sphere → sub-space` means the briefing always matches the current context.
A file in `library/notes/threads/` gets a different orientation than one in
`workshop/commissions/`.

**Memory is sphere-local** — The Scribe tracks threads and patterns. The Tavern Keeper
tracks relationships. Keeping memory local prevents cross-contamination between spheres.
The Oracle in the Woodlands doesn't need to know about research commissions.

**Skills are sphere-scoped** — Claude Code auto-discovers `.claude/commands/` relative to
wherever you're working, so a skill placed in a sphere's `.claude/commands/` is invokable
whenever you're anywhere inside that sphere's tree (e.g. `/advisor-a` from anywhere in
`sp_academy/`). A skill that needs to scan or act across every sphere belongs in `bag/`,
the cross-sphere utility layer, rather than in any one sphere.

**Agent files are ephemeral** — The `.jsonl` session logs are not designed by hand.
The human-authored layer — CLAUDE.md + memory files + skill files — is what persists
and carries meaning between sessions. Agent logs are a trace, not a structure.

## The Four File Types Compared

| File type | Who writes it | Scope | Persists? |
|-----------|--------------|-------|-----------|
| CLAUDE.md | Human | Sphere or sub-space | Yes — these are the system |
| Memory files | Claude (on departure) | Sphere-local | Yes — updated each session |
| Skill files | Human | Sphere-scoped (or bag/ if cross-sphere) | Yes — invocable rituals |
| Agent files | Claude Code (auto) | Session | Ephemeral — logs only |
